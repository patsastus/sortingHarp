# `sortingHarp`: sonify your sorting function

## The idea
I'm currently learning C++, and was tasked with writing a Ford-Johnson merge-insert sorting algorithm. It is concerned with minimizing the number of comparisons needed to sort an list of numbers, so for an audio hackathon I thought it would be interesting to hear those comparisons happening, and created this program. It does a single modification to the sorting algorithm: it adds the `WAVFILE` line to the overloaded comparison operator I made to keep track of the number of comparisons:
```
bool PMergeMe::vectorNode::operator<(vectorNode const &other) const{
     ++gComparisons;  //count comparisons
    WAVFILE << this->value << other.value;
    return this->value < other.value;
}
```

## Making sounds from numbers
Sounds *are* just numbers in a row written to a .wav file, but it's not quite as simple as that single line of code makes it seem. Behind the `WAVFILE` macro lives a class that takes each number, and the information of the minimum and maximum of the set being sorted, and maps the number to a tone on the pentatonic scale, with the lowest value mapping to a **C1**, and the highest to a **C6**, so a map of 26 notes. 

We're always working with a pair of numbers being compared: the left-hand side of the comparison gets written to the left audio channel, and the right-hand side gets written to the right. There's a function to translate the tone frequencies to samples of a 50-millisecond sine wave with a simple triangular envelope to give a somewhat pleasant sound out of it.

## Running `sortingHarp`
```
make
./sortingharp <sequence of numbers>
```
this will generate a sound file called `ford_johnson_sort.wav`. The length will vary both depending on input length and how sorted the input was to start with. Each comparison made adds a 50-millisecond burst of sound, and the algorithm ensures that at most 
$$\sum_{k=1}^{n}\lceil log_{2} \left( \frac{3k}{4} \right) \rceil$$
comparisons are made, where `n` is the length of the input sequence. In practice, a 350-number input results in around 2400 comparisons (worst-case allowance 2473), which corresponds to a  roughly 2-minute audio file.

By default the program creates a `ford_johnson_sort.wav` as well as  a `standard_sort.wav` file, but you can capture the sound of any sorting algorithm (so long as it uses `operator<`).

## Running `render_video.py`
This script generates a video visualization of the merge-insert sort algorithm. This was vary quickly vibecoded to give a visual element to a audio project, so don't expect much. It required adding some state printouts inside the sorting algorithm, to have the current state to work with to build the proper images. 
### Prerequisites
- Python 3.12+
- [uv](https://github.com/astral-sh/uv) package manager
- ffmpeg
### Setup
```bash
# Install uv if you haven't already
curl -LsSf https://astral.sh/uv/install.sh | sh
# Sync dependencies
uv sync
uv run render_video.py
```
This will:
1. Load `events.json` (sorting event data generated by `./sortingHarp <number-sequence>)
2. Render `sorting_video_silent.mp4`
3. If `ford_johnson_sort.wav` exists, combine it into `sorting_video.mp4`
